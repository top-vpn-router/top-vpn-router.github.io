AmneziaVPN — это не коммерческий сервис, а клиент-сервер с открытым кодом. Вы арендуете виртуальный сервер за 150 ₽/мес, устанавливаете на него Amnezia-пакет и получаете личный VPN с VLESS, AmneziaWG (модифицированный WireGuard) и Shadowsocks. Под Keenetic собран специальный модуль — интегрируется в KeeneticOS как родной VPN-клиент. На OpenWrt есть готовый ipk-пакет.

Плюсы
  • Свой сервер = свои логи (они есть только у вас)
  • AmneziaWG маскируется под случайный трафик — ТСПУ не распознаёт

  • Нет ежемесячной подписки — только аренда VDS (от 150 ₽)
Минусы
  • Нужно разбираться с арендой VDS — новичкам сложно
  • Один сервер — одна страна: несколько «локаций» = несколько серверов
  • Поддержки как у коммерческого сервиса нет — только GitHub и Telegram

03

Как установить и настроить VPN на роутере — инструкции по брендам

Три самые популярные модели в России 2026 года. Шаги отличаются, но логика одна: скачать конфиг-файл у VPN-сервиса → загрузить его в роутер → активировать соединение.

Прошивка: KeeneticOS 3.7 или новее (проверить: «Общие настройки» → «Обновления»). Для VLESS — установите из Amnezia repo модуль KeeneticOS Amnezia.
01

Установите компонент VPN-клиента

Откройте веб-интерфейс по адресу my.keenetic.net или 192.168.1.1. Перейдите в «Общие настройки» → «Обновления и компоненты» → «Изменить набор компонентов». В поиске наберите WireGuard или OpenVPN — поставьте галочку и сохраните. Роутер перезагрузится.

02

Получите конфиг-файл у VPN-сервиса

В личном кабинете VPN-сервиса найдите раздел «Настройка на роутере» или «Manual configuration». Скачайте .conf (WireGuard) или .ovpn (OpenVPN) файл. У VPNTYPE, HideMy.Name и Mullvad файл даётся сразу с нужной страной и учётными данными.

03

Создайте новое соединение

В интерфейсе Keenetic: «Интернет» → «Другие подключения» → «Добавить соединение». Выберите тип (WireGuard / OpenVPN), загрузите скачанный конфиг-файл. Введите логин и пароль, если сервис их запрашивает.

04

Проверьте подключение

Рядом с новым соединением должен загореться зелёный индикатор. В «Диагностике» → «Система» появится новый интерфейс nwg0 (WireGuard) или ovpn_br0 с отправленным и принятым трафиком.

05

Направьте трафик через VPN

«Приоритеты подключений» — перетащите VPN-соединение выше основного провайдера. Либо зайдите в «Сетевые правила» → «Маршрутизация» и задайте маршрут для конкретных устройств (умный ТВ, приставка) через VPN, остальные — напрямую.

✓ Проверка: с любого устройства в домашнем Wi-Fi откройте 2ip.ru — IP и страна должны поменяться на адрес VPN-сервера. Если нет — в настройках соединения уберите галочку «Использовать для выхода в интернет только по этому подключению», сохраните, включите обратно.
Прошивка: штатная 1.2.x для Archer AX73 / AX55 / C7 v5. На Archer C6 v3 — OpenWrt 22.03 или новее. WireGuard на штатной прошивке отсутствует.
01

Зайдите в веб-интерфейс роутера

Адрес — tplinkwifi.net или 192.168.0.1. Логин и пароль — с наклейки на роутере (обычно admin/admin или admin/пароль с коробки).

02

Откройте раздел VPN Client

«Дополнительные настройки» → «VPN Client». На некоторых моделях раздел называется «VPN Server/Client». Если его нет — прошивка старая, обновите на сайте tp-link.com/support → введите модель → Firmware.

03

Загрузите OpenVPN-профиль

Нажмите «Добавить». Тип — OpenVPN. Название — произвольное. Загрузите .ovpn файл от VPN-сервиса. Введите логин и пароль, если они запрашиваются отдельно.

04

Добавьте устройства в VPN

На TP-Link по умолчанию через VPN идут не все, а только указанные устройства. В разделе «VPN Client» → «Add» → укажите MAC-адреса (из «Карта сети» → «Подключённые устройства») или выберите сеть целиком.

05

Активируйте подключение

Переключатель рядом с профилем — в положение «Включено». Статус должен смениться на «Connected». Если соединение падает — проверьте в логах, что порт UDP (обычно 1194 для OpenVPN) не блокируется провайдером.

✓ Проверка: откройте 2ip.ru с добавленного в VPN устройства — IP должен быть из страны VPN. Если с одних устройств работает, с других нет — значит, не все MAC-адреса добавлены в список VPN-клиента.
Прошивка: ASUSWRT 388.x для нативного WireGuard. Для продвинутых возможностей (kill switch, маршрутизация по правилам) — Asuswrt-Merlin 388.7+. Поддерживает RT-AX86U, AX88U, AC86U, AX68U.
01

Зайдите в ASUSWRT

Адрес — router.asus.com или 192.168.1.1. В левом меню — раздел «VPN».

02

Выберите VPN Client → Add profile

Вкладка «VPN Client» → кнопка «Add profile». Появятся варианты: PPTP, L2TP, OpenVPN, WireGuard. Для новых роутеров выбирайте WireGuard — быстрее и стабильнее.

03

Импортируйте конфиг-файл

Кнопка «Import .ovpn file» или «Import .conf file». Загрузите файл от VPN-сервиса. Имя профиля — любое, например «VPN_Home». Сохраните.

04

Активируйте и настройте правила

Переключатель «Activate» рядом с профилем. В «Rules» задаётся, какие устройства или диапазоны IP идут через VPN, а какие напрямую — полезно, если умный ТВ должен оставаться в локальной сети провайдера.

05

Включите kill switch (опционально)

В Merlin-прошивке — галочка «Block routed clients if tunnel goes down». Если VPN разорвётся, устройства из правила не получат интернет через обычное подключение — утечки реального IP не будет.

✓ Проверка: статус профиля — «Connected». На устройстве в VPN-сегменте — 2ip.ru показывает новый IP. В разделе «VPN Client» → «Statistics» должен расти приём/отправка в реальном времени.
Прошивка: OpenWrt 22.03 или 23.05. Устанавливается на Xiaomi AX3000T/AX3200, TP-Link C6 v3, D-Link DIR-878, Netgear R7800 и десятки других. Перед прошивкой — проверка модели и ревизии на openwrt.org/toh.
01

Установите нужный пакет

Подключитесь к роутеру по SSH (ssh root@192.168.1.1) или через LuCI → System → Software. Для WireGuard: opkg install wireguard-tools luci-app-wireguard. Для OpenVPN: opkg install openvpn-openssl luci-app-openvpn.

02

Создайте VPN-интерфейс в LuCI

Network → Interfaces → Add new interface. Название — wg0 для WireGuard или ovpn для OpenVPN. Protocol — соответствующий. Для WireGuard сразу вставьте в интерфейс ключи и данные пиров из конфига сервиса.

03

Настройте firewall

Network → Firewall → добавить зону vpn. Forwarding — From LAN to VPN. Masquerading — включить. Без этого шага трафик от устройств в LAN не уйдёт в туннель.

04

Укажите DNS VPN-сервиса

В интерфейсе VPN задайте DNS 10.2.0.1 (у большинства сервисов) или конкретный от VPN-провайдера. Это обязательно — иначе DNS-запросы утекают через провайдера, и блокировки остаются.

05

Поднимите интерфейс

«Save & Apply» — VPN-интерфейс поднимается. В Status → Interfaces должен быть RX/TX ≠ 0. Для VLESS/XRAY на OpenWrt ставят пакет xray-core вручную — сложнее, но работает на слабых SoC (MT7621, IPQ8072).

✓ Проверка: curl ipinfo.io с роутера по SSH — вернёт IP VPN-сервера. Если вернул обычный IP провайдера — смотрите iptables -t nat -L -v: нет правил MASQUERADE → не настроен firewall.

Типичные ошибки — и как их быстро исправить

  • Соединение «подключено», но IP не сменился. В маршрутизации не задан default gateway через VPN. На Keenetic — «Приоритеты подключений», VPN выше. На OpenWrt — настройки firewall и masquerading.
  • DNS-утечка. Устройства получают DNS от провайдера, минуя туннель. Пропишите в интерфейсе VPN DNS от сервиса (обычно 10.2.0.1 или конкретный IP в конфиге).
  • Падает скорость на 70–80%. Роутеру не хватает CPU для шифрования. Помогает смена OpenVPN на WireGuard (в 2–3 раза быстрее). Если это не помогло — роутер слабый, CPU ≤600 МГц.
  • Периодический разрыв туннеля. Сервер сервиса перегружен или провайдер блокирует по IP. Смените точку входа в конфиге на другую страну — обычно помогает.

Как работает VPN на роутере — и почему это удобнее, чем ставить приложение

Короткое техническое объяснение, которое пригодится при настройке и выборе тарифа.

→

Один туннель на весь дом

Роутер поднимает VPN-туннель и делает всё устройства в квартире «выходящими в интернет» через него. Телевизор, PlayStation, умная колонка, ноутбук — все получают новый IP и минус одну блокировку. На каждом устройстве ставить приложение не нужно.

→

Лицензия на роутер — это одна «точка»

С точки зрения VPN-сервиса роутер — это одно устройство, даже если через него работают десять домашних гаджетов. Вместо пяти подписок на семью вы оплачиваете одну на роутер. Для домов с Android TV и IoT это особенно выгодно.

→

Устройства без поддержки VPN тоже защищены

Smart-TV, приставка, колонка, IoT-розетки, игровая консоль — многие не умеют работать с VPN-приложениями. Через роутер они автоматически попадают в туннель. Это единственный рабочий способ защитить такие устройства.

→

Трафик не проходит через «облачных» посредников

В мобильных приложениях VPN часто идут через сторонние SDK-серверы — это точка утечки метаданных. Туннель на роутере — прямой, от вашего роутера до VPN-сервера. Меньше звеньев — выше приватность.

Обратная сторона: если VPN-сервер перегружен или ловится ТСПУ — «падает» весь домашний интернет, не только одно приложение. Частичное решение — правила маршрутизации: через VPN идут только те устройства, которым это нужно (приставка, ПК), а умный ТВ из российского каталога остаётся на прямом подключении.

  • Кто не хочет ставить VPN-приложение на каждое устройство
  • Кто использует сервисы, требующие стабильный IP в другой стране

✗ Кому лучше приложение на устройстве

  • Если VPN нужен только в браузере — проще поставить расширение
  • Если часто меняются сервер — на роутере это неудобно
  • Если роутер арендный от провайдера, без доступа к прошивке
  • Если есть ноутбук, который работает и дома, и вне дома
08

Что учесть при выборе VPN для роутера

Пять практических критериев, которые решают — будет VPN работать у вас дома или разочарует.

A

Протокол важнее бренда сервиса

Если у провайдера ловится OpenVPN — не поможет даже самый раскрученный бренд. В российских сетях 2026 года стабильно работают VLESS, AmneziaWG и Shadowsocks. WireGuard — быстрый, но его всё чаще замедляют. Перед подпиской уточняйте у сервиса, поддерживает ли он эти протоколы и даёт ли конфиги для вашего роутера.

VLESS, AmneziaWG — обходят ТСПУ OpenVPN — замедляется PPTP — устарел
B

Процессор роутера — потолок скорости

Шифрование VPN нагружает CPU роутера. Старый двухъядерный 600 МГц Keenetic Giga II — максимум 40–50 Мбит/с через WireGuard и 15–20 через OpenVPN. Если провайдерский канал 300 Мбит/с, вы не увидите больше 50 — это ограничение железа. Для полноценного использования нужен роутер с CPU 1 ГГц+ и аппаратным ускорением шифрования.

1 ГГц+ CPU — скорость до 150 Мбит/с ≤600 МГц — до 50 Мбит/с
C

Маршрутизация по устройствам — не роскошь, а необходимость

Пустить весь дом через VPN — привлекательная идея, но умный ТВ с российским сервисом (Кинопоиск, Okko, ivi) работать через зарубежный VPN не будет. Решение — правила маршрутизации: через VPN идёт только компьютер, приставка с Netflix и смартфон, а ТВ остаётся на прямом подключении. Эту функцию поддерживают Keenetic, Asus Merlin, OpenWrt и GL.iNet.

Keenetic, Asus Merlin, OpenWrt — поддерживают TP-Link штатно — только полная маршрутизация
D

Kill switch на роутере работает надёжнее, чем в приложении

Если VPN-туннель падает, устройства без kill switch автоматически уходят на обычное подключение — утечка реального IP гарантирована. Kill switch на роутере (правило firewall: блокировать всё, кроме VPN-интерфейса) отрабатывает на уровне железа. У Asus Merlin и OpenWrt настраивается одной галочкой. У Keenetic — через правила маршрутизации с приоритетом «Только по этому соединению».

E

Бесплатные VPN — на роутер не ставить

Большинство бесплатных сервисов не дают конфигурационные файлы для ручной настройки — их бизнес-модель именно в приложении, которое собирает телеметрию. Если «бесплатный» сервис всё-таки даёт конфиг, проверяйте его политику данных. Исключения: open-source (AmneziaVPN + свой VDS), Windscribe (10 ГБ/мес честно), ProtonVPN (строгая швейцарская юрисдикция).

AmneziaVPN + VDS — свой сервер Windscribe free 10 ГБ Бесплатные без конфига — не для роутера
09

Вопросы и ответы о VPN для роутера

Собрал десять самых частых вопросов, которые мне задают в комментариях и на форумах Keenetic и OpenWrt.

Для роутеров Keenetic, Asus и OpenWrt в июле 2026 года стабильно работают VPNTYPE (VLESS, WireGuard), AdGuard VPN (OpenVPN + DNS-фильтр), HideMy.Name (WireGuard и OpenVPN конфиги, работает в РФ с 2014 года), Surfshark (готовая прошивка для Asus Merlin) и AmneziaVPN (свой сервер). Выбор зависит от модели — подробно в разделе «Рейтинг».

Нет. VPN-клиент работает только на роутерах с соответствующей прошивкой. Keenetic, Asus RT-AX, GL.iNet, MikroTik — поддерживают VPN из коробки. TP-Link и D-Link — частично (OpenVPN), для WireGuard нужна OpenWrt. Xiaomi и Huawei в штатной прошивке VPN-клиента не имеют. Перед покупкой или настройкой проверяйте: точная модель + ревизия на openwrt.org/toh для сторонней прошивки, либо инструкции производителя для родной.

Для Keenetic Hopper, Viva, Giga: my.keenetic.net → «Общие настройки» → «Обновления и компоненты» → включите «Клиент WireGuard» или «Клиент OpenVPN». Роутер перезагрузится. Далее: «Интернет» → «Другие подключения» → «Добавить» → выбрать тип, загрузить конфиг-файл от VPN-сервиса. Подробнее — в разделе «Установка».

На Archer AX73/AX55/C7: tplinkwifi.net → «Дополнительные настройки» → «VPN Client» → «Добавить». Выберите OpenVPN, загрузите .ovpn-файл. Для WireGuard потребуется прошивка OpenWrt — проверяйте совместимость модели и ревизии. На OpenWrt: LuCI → Network → Interfaces → Add → Protocol WireGuard, вставить ключи из конфига.

ASUSWRT 388.x умеет WireGuard из коробки. router.asus.com → «VPN» → «VPN Client» → «Add profile» → выберите WireGuard или OpenVPN. Импортируйте .conf или .ovpn файл, сохраните. Переключатель «Activate» — и трафик уходит через VPN. Для продвинутой маршрутизации по устройствам и kill switch — поставьте Asuswrt-Merlin поверх родной прошивки.

Полностью бесплатные VPN для роутера — редкость. Рабочие варианты: AmneziaVPN (open-source + свой VDS-сервер на Timeweb или Aéza от 150 ₽/мес), Windscribe (10 ГБ/мес бесплатно при регистрации по email, даёт WireGuard и OpenVPN-конфиги), Planet VPN (OpenVPN файлы на бесплатном тарифе — 5 серверов). ProtonVPN free даёт ограниченно страны. Совсем бесплатные «мобильные» VPN конфигов для роутера обычно не дают — они монетизируются через телеметрию.

В 2026 году для дома: Keenetic Hopper KN-3810 (от 5 900 ₽) — оптимальный, VPN из коробки плюс VLESS через модуль Amnezia. TP-Link Archer AX73 (от 8 400 ₽) — для OpenVPN. Asus RT-AX86U Pro (от 18 500 ₽) — WireGuard нативно, поддержка Merlin. Xiaomi AX3000T (от 4 200 ₽) — бюджет после перепрошивки на OpenWrt. GL.iNet GL-MT6000 (от 13 900 ₽) — если VPN главная задача.

На штатной прошивке Xiaomi VPN-клиента нет. Для работы потребуется перепрошивка на OpenWrt или PandoraBox. Известно совместимы: Mi Router AX3000T, AX3200, AX6000, Redmi AX6S. Процедура сложнее, чем на TP-Link — требует разблокировки bootloader и загрузки образа через serial-консоль на некоторых моделях. Перепрошивка отменяет гарантию. Подробные гайды — на openwrt.org/toh/xiaomi и 4pda.

На KeeneticOS 3.7+ откройте «Общие настройки» → «Обновления и компоненты» → нажмите на кнопку «Обновить список репозиториев», затем добавьте репозиторий Amnezia (инструкция есть в документации проекта на GitHub). После этого среди доступных компонентов появится «Amnezia Client» — установите. В «Интернет» → «Другие подключения» будет тип AmneziaWG или VLESS. Загрузите конфиг, полученный от вашего сервера AmneziaVPN.

С любого устройства, подключённого к Wi-Fi роутера, откройте 2ip.ru — ваш IP и страна должны стать адресом VPN-сервера. Для утечки DNS — dnsleaktest.com → Extended Test: все DNS из страны VPN. Для IPv6 — ipv6leak.com. В веб-интерфейсе роутера (Keenetic, ASUSWRT) в разделе VPN-клиент должен гореть индикатор «Connected» и расти счётчик RX/TX. Если IP не поменялся — проверьте «Приоритеты подключений» (Keenetic) или правила firewall/masquerading (OpenWrt).
